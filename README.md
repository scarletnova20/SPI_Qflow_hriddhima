# SPI Controller Physical Design using Qflow

This repository contains the complete RTL-to-GDSII physical design flow for an SPI Master Controller. The project was implemented using the open-source Qflow digital synthesis flow targeting the OSU018 (0.18um / 180nm SCMOS) standard cell library.

## Project Overview

The objective of this project is to demonstrate a full ASIC physical design flow, encompassing synthesis, placement, routing, static timing analysis (STA), and design rule checking (DRC) for an industry-standard SPI design. 

The design sources are based on the OpenCores Wishbone SPI controller, integrated and synthesized using the Qflow toolchain on an Ubuntu environment (via WSL).

## Tools and Technology

*   **PDK:** OSU018 (0.18um 6-metal SCMOS)
*   **Flow Manager:** Qflow 1.4.104
*   **Synthesis:** Yosys 0.33
*   **Placement:** GrayWolf / TimberWolfSC
*   **Routing:** Qrouter 1.4.71
*   **Layout & DRC:** Magic VLSI

## Repository Structure

*   `/source`: Contains the Verilog RTL source files (`spi_top.v`, `spi_shift.v`, `spi_clgen.v`, `spi_defines.v`, `timescale.v`).
*   `/synthesis`: Contains the synthesized netlists (`spi_top.v`, `spi_top.spc`) generated by Yosys.
*   `/layout`: Contains the physical design outputs including the DEF files (`spi_top.def`), Magic layout (`spi_top.mag`), and the final GDSII mask data (`spi_top.gds`).
*   `/logs`: Contains the execution logs for each stage of the Qflow pipeline (synthesis, placement, routing, STA, DRC).
*   `/docs`: Contains screenshots capturing the tool GUIs and flow progress.

## Physical Design Steps

The project was executed through the following sequential stages:

### 1. Synthesis
The RTL code was synthesized using Yosys to map the behavioral logic to the OSU018 standard cell library.
*   **Result:** The design was mapped to 2,709 standard cells (including 253 D-Flip Flops). High fanout nets were buffered appropriately.

### 2. Placement
GrayWolf was utilized for standard cell placement. The cells were arranged into rows, and power/ground stripes (2.0um width, 50.0um pitch) were added to ensure proper power distribution.

### 3. Routing
Qrouter performed maze routing across the standard cells using the available metal layers defined by the OSU018 tech file. 
*   **Result:** Successfully routed 7,644 nets across three stages with 100% completion and zero failed routes.

### 4. Static Timing Analysis (STA)
Post-route static timing analysis was conducted using the integrated OpenSTA engine to verify hold and setup times based on the extracted parasitics.

### 5. Layout and DRC
The DEF files were migrated to Magic for layout generation and Design Rule Checking. The layout was verified against the SCN6M_SUBM.10 tech rules. 

### 6. GDSII Generation
Following a successful layout check, the final mask data was exported in the industry-standard GDSII format (`spi_top.gds`).

## Execution and Reproduction

To reproduce this flow:
1. Ensure Qflow, Yosys, GrayWolf, Qrouter, and Magic are installed on a Linux environment.
2. Initialize the Qflow project utilizing the OSU018 technology directory.
3. Place the Verilog files in the project `source` directory.
4. Execute the flow sequentially via the Qflow GUI manager or command-line interface.
